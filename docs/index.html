Guida d'uso

Generatori per il tuo gioco fantasy, in una riga

fantasy-game-toolkit è una libreria Java di piccoli generatori — personaggi e i loro nomi, gioielli, armi, armature, pozioni, buff e debuff — pensati per un'unica cosa: darti un risultato pronto con una sola catena di chiamate.

Non c'è configurazione, non ci sono file da preparare, non c'è stato da gestire. Ogni generatore si usa sempre allo stesso modo: parti da building(), aggiungi cosa vuoi, chiudi con generate(). Impari un generatore e li sai usare tutti.

In una riga: JewelGeneratorTool.building().randomJewel().randomRarity().generate(); — e hai un gioiello completo di rarità e bonus. Stessa identica forma per gli altri tool.

Lo schema comune

Tutti i generatori del toolkit seguono lo stesso contratto. Una volta capito questo, ogni tool è immediato:

  • La classe si chiama XxxTool ed espone un solo punto d'ingresso statico: building().
  • building() restituisce un Builder fluente: aggiungi le opzioni che ti servono, in qualsiasi ordine.
  • La catena si chiude con generate(), che restituisce un result immutabile (un record). L'unica eccezione è il lancio di dadi, che chiude con roll().
  • Ogni result implementa la marker interface comune GeneratedElementResult.
Sempre questa forma
XxxResult result = XxxGeneratorTool.building()
        .opzione(...)      // zero o più opzioni, in qualsiasi ordine
        .generate();       // chiude e restituisce un record immutabile
La generazione è casuale e non riproducibile. Ogni tool usa internamente un new Random(): non c'è un seme da passare, quindi due chiamate identiche possono dare risultati diversi. Se un input obbligatorio manca o è incoerente, generate() lancia IllegalStateException con un messaggio diagnostico.

Installazione

Progetto Maven, Java 21. Coordinate dell'artefatto:

pom.xml
<dependency>
    <groupId>it.fantasytoolkit</groupId>
    <artifactId>fantasytoolkit</artifactId>
    <version>1.0-SNAPSHOT</version>
</dependency>

A runtime non servono dipendenze esterne: le liste di nomi sono file .txt sul classpath. Lombok è usato solo a compile-time (scope provided).

Il modello condiviso

I generatori parlano un vocabolario comune di tipi di dominio (package it.fantasytoolkitcore.core.model). Li ritrovi ovunque:

TipoValoriNote

RarityCOMMON, UNCOMMON, RARE, EPIC, LEGENDARYOrdine significativo (dal più comune al più raro).
JewelRING, NECKLACE, BRACELET, EARRINGTipi di gioiello.
WeaponSWORD, BOW, AXE, HAMMER, SHIELD, DAGGER, CROSSBOW, BATTLEAXE, STAFF, SCEPTERTipi di arma.
ArmourCHESTPLATE, PANTS, BOOTS, GAUNTLETS, HELMET, BELTTipi di armatura.

C'è poi RarityTable, una tabella di estrazione pesata delle rarità (usata dai gioielli). Si costruisce col builder e la somma dei pesi deve fare 100:

RarityTable
RarityTable table = RarityTable.builder()
        .entry(Rarity.COMMON, 60)
        .entry(Rarity.UNCOMMON, 20)
        .entry(Rarity.RARE, 19)
        .entry(Rarity.LEGENDARY, 1)   // somma pesi = 100
        .build();

Se ti serve una distribuzione già pronta senza costruire una tabella a mano, usa RaritySelector: incapsula una RarityTable con i pesi standard e restituisce direttamente una Rarity con select(Random).

RaritàProbabilità (default)
COMMON50%
UNCOMMON25%
RARE12%
EPIC8%
LEGENDARY5%
RaritySelector
Random random = new Random();
Rarity rarity = RaritySelector.withDefaultDistribution().select(random);

// la rarità così estratta si passa ai generatori
WeaponGeneratorTool.building().randomWeapon().rarity(rarity).generate();
La RarityTable interna a RaritySelector non è esposta: il canale per alimentare i generatori è select(random) → rarity(Rarity).

🧝 Nomi di personaggio

CharacterNameGeneratorTool genera un nome pescandolo dalla lista associata alla razza. È il generatore più semplice: l'unica cosa obbligatoria è la razza.

Nome semplice
NameResult result = CharacterNameGeneratorTool.building()
        .race(Race.ORC)
        .generate();

result.name();   // es. "Gorbash"

Con addNickname() al nome di razza viene aggiunto un soprannome (da nicknames.txt):

Nome + soprannome
// i metodi sono fluenti e in qualsiasi ordine
NameResult result = CharacterNameGeneratorTool.building()
        .addNickname()
        .race(Race.HUMAN)
        .generate();

result.name();   // es. "Aldric il Coraggioso"
Metodo del builderA cosa serveObbligatorio
race(Race)Sceglie la lista di nomi in base alla razzaSì
addNickname()Aggiunge un soprannome dopo il nomeNo
generate()Produce il NameResult—

Risultato: NameResult(String name). Senza race(...) → IllegalStateException.

💍 Gioielli

JewelGeneratorTool genera un gioiello completo: tipo, rarità e i relativi bonus (buff/debuff). Devi scegliere esattamente una fonte per il tipo e una per la rarità — il resto lo fa il tool.

Casi tipici
// tipo fisso + rarità fissa
JewelGeneratorTool.building().jewel(Jewel.RING).rarity(Rarity.EPIC).generate();

// tipo casuale + rarità casuale fino a un massimo
JewelGeneratorTool.building().randomJewel().maxRarity(Rarity.RARE).generate();

// tipo fisso + rarità pescata da una tabella pesata
JewelGeneratorTool.building().jewel(Jewel.NECKLACE).rarityTable(table).generate();

// tutto casuale
JewelGeneratorTool.building().randomJewel().randomRarity().generate();

Scegliere il tipo di gioiello

Esattamente una tra queste (nessuna o entrambe → IllegalStateException):

MetodoEffetto
jewel(Jewel)Tipo fisso
randomJewel()Casuale tra tutti i Jewel.values()

Scegliere la rarità

Esattamente una tra queste (zero o più di una → IllegalStateException):

MetodoEffetto
rarity(Rarity)Rarità fissa
maxRarity(Rarity)Casuale fino a quel livello incluso
rarityTable(RarityTable)Estrazione pesata dalla tabella
randomRarity()Casuale tra tutte le rarità

Bonus del gioiello

Di default il gioiello riceve automaticamente i suoi bonus, calibrati sulla rarità risolta (dietro le quinte usa il generatore di Buff & Debuff). Se vuoi un gioiello "nudo", senza alcun effetto, aggiungi noStatusEffect():

Gioiello senza bonus
JewelResult result = JewelGeneratorTool.building()
        .jewel(Jewel.RING)
        .rarity(Rarity.COMMON)
        .noStatusEffect()        // buffs e debuffs vuoti
        .generate();

result.buffs();     // [] vuota
result.debuffs();   // [] vuota
noStatusEffect() incide solo sui bonus: tipo e rarità restano quelli che hai scelto, ed è combinabile con qualsiasi fonte di tipo/rarità.

⚔️ Armi

WeaponGeneratorTool è la copia esatta del generatore di gioielli: stessa sintassi, stesse regole. Cambia solo il tipo generato. Scegli esattamente una fonte per il tipo e una per la rarità; i bonus arrivano di default e li puoi togliere con noStatusEffect(). In più, ogni arma ha un valore di attacco (attack) pescato casualmente in un intervallo che dipende dalla rarità.

Casi tipici
// arma fissa + rarità fissa
WeaponGeneratorTool.building().weapon(Weapon.SWORD).rarity(Rarity.EPIC).generate();

// arma casuale + rarità casuale fino a un massimo
WeaponGeneratorTool.building().randomWeapon().maxRarity(Rarity.RARE).generate();

// senza bonus
WeaponGeneratorTool.building().weapon(Weapon.BOW).rarity(Rarity.COMMON).noStatusEffect().generate();

Tutte le opzioni del builder

Il valore di attacco

L'attacco è pescato a caso nell'intervallo associato alla rarità da WeaponRules. La regola di default (DefaultWeaponRules) è deliberatamente semplice — un intervallo [min–max] per rarità:

RaritàAttacco (default)
COMMON1 – 3
UNCOMMON3 – 6
RARE6 – 10
EPIC10 – 15
LEGENDARY15 – 25

Vuoi valori diversi? Passa una tua WeaponRules (interfaccia gemella di BuffDebuffRules):

Attacco con regole custom
// attacco sempre 50, per ogni rarità
WeaponRules customRules = rarity -> new AttackRange(50, 50);

WeaponResult result = WeaponGeneratorTool.building()
        .weapon(Weapon.SWORD)
        .rarity(Rarity.EPIC)
        .rules(customRules)
        .generate();

result.attack();   // 50

Risultato: WeaponResult(Weapon weapon, Rarity rarity, List<BuffElement> buffs, List<DebuffElement> debuffs, int attack). Le regole di validazione (fonte mancante o doppia) sono identiche a quelle dei gioielli.

🛡️ Armature

ArmourGeneratorTool.building().randomArmour().rarityTable(table).generate();

// senza bonus

UNCOMMON2 – 4
RARE4 – 7

🧪 Pozioni

PotionGeneratorTool usa lo stesso builder degli altri tool, ma qui scegli una famiglia (PotionType) invece di un tipo d'oggetto. A seconda della famiglia, la pozione porta un contenuto diverso — un buff, un debuff, oppure un valore di rigenerazione — mentre la rarità si sceglie con le solite quattro fonti.

Casi tipici
// pozione di buff, rarità fissa
PotionGeneratorTool.building().type(PotionType.BUFF).rarity(Rarity.EPIC).generate();

// pozione di debuff, rarità casuale fino a un massimo
PotionGeneratorTool.building().type(PotionType.DEBUFF).maxRarity(Rarity.RARE).generate();

// pozione di rigenerazione vita
PotionGeneratorTool.building().type(PotionType.HEALTH_REGENERATION).rarity(Rarity.COMMON).generate();

// tutto casuale
PotionGeneratorTool.building().randomType().randomRarity().generate();

Scegliere la famiglia

Esattamente una tra queste (nessuna o entrambe → IllegalStateException):

MetodoEffetto
type(PotionType)Famiglia fissa
randomType()Casuale tra tutti i PotionType.values()

La rarità si sceglie con le stesse quattro fonti (e le stesse regole di validazione) dei gioielli: rarity, maxRarity, rarityTable, randomRarity.

Cosa contiene la pozione

Il contenuto dipende dalla famiglia risolta:

FamigliaContenuto
BUFFUn singolo BuffElement (dietro le quinte usa il generatore di Buff & Debuff). value = 0, debuff = null.
DEBUFFUn singolo DebuffElement. value = 0, buff = null.
HEALTH_REGENERATION · MANA_REGENERATIONUn value intero (punti vita/mana rigenerati), calibrato sulla rarità. buff/debuff = null.

Il valore di rigenerazione

Per le pozioni di rigenerazione, il value è pescato nell'intervallo associato alla rarità da PotionRules (default DefaultPotionRules). Stesso intervallo per vita e mana — a distinguerle è solo il PotionType:

RaritàRigenerazione (default)
COMMON5 – 10
UNCOMMON10 – 20
RARE20 – 35
EPIC35 – 55
LEGENDARY55 – 90

Range personalizzabili con rules(PotionRules), come per l'attacco delle armi (rarity -> new RegenerationRange(min, max)).

I debuff delle pozioni sono derivati da un buff. Dato che BuffDebuffGeneratorTool non genera ancora debuff reali, la pozione DEBUFF genera un BuffElement e ne riusa characteristic e value per costruire un DebuffElement. Quando il generatore di buff/debuff produrrà debuff propri, questo si potrà adeguare.
Le pozioni non hanno noStatusEffect(): il contenuto (buff, debuff o rigenerazione) è la ragione d'essere della pozione, non un extra opzionale.

🧙 Personaggi

CharacterGeneratorTool mette insieme un personaggio completo: razza, classe, nome e caratteristiche con i punti distribuiti. Non usa rarità né status effect come gli oggetti, ma applica alle caratteristiche i bonus di razza e di classe. Stesso builder fluente degli altri tool.

Casi tipici
// razza e classe fisse, tutte le caratteristiche
CharacterGeneratorTool.building().race(Race.ELF).characterClass(CharacterClass.MAGE).allCharacteristics().totalPoints(30).generate();

// tutto casuale, con soprannome
CharacterGeneratorTool.building().randomRace().randomClass().addNickname().allCharacteristics().totalPoints(50).generate();

// senza bonus di razza né di classe (opt-out con tabelle vuote)
CharacterGeneratorTool.building().race(Race.HUMAN).characterClass(CharacterClass.THIEF)
        .characteristics(List.of(Characteristic.STRENGTH)).totalPoints(1)
        .raceBonusTable(RaceBonusTable.builder().build())
        .classBonusTable(ClassBonusTable.builder().build()).generate();

Razza, classe e caratteristiche

Razza e classe funzionano allo stesso modo: per ognuna scegli esattamente una fonte (nessuna o entrambe → IllegalStateException). Ogni personaggio ha sempre una razza e una classe.

MetodoEffetto
race(Race)Razza fissa
randomRace()Casuale tra tutti i Race.values()
characterClass(CharacterClass)Classe fissa
randomClass()Casuale tra tutti i CharacterClass.values()

Le caratteristiche si scelgono con esattamente una tra characteristics(List<Characteristic>) (lista esplicita, de-duplicata preservando l'ordine) e allCharacteristics() (tutte le Characteristic.values()). Nessuna o entrambe → IllegalStateException. Il nome è generato internamente riusando il generatore di nomi con la razza risolta; addNickname() aggiunge il soprannome.

I punti e la distribuzione

totalPoints(int) è obbligatorio (altrimenti IllegalStateException): è il monte punti totale distribuito tra le caratteristiche. minCharacteristicValue(int) è il minimo garantito a ognuna (default 1, non negativo). Se totalPoints < min * numeroCaratteristiche → eccezione diagnostica.

Ogni caratteristica parte dal minimo, poi i punti residui vengono assegnati uno alla volta a caratteristiche casuali. La somma dei valori dopo la sola distribuzione è sempre esattamente totalPoints.

I bonus di razza e classe

Come ultimi passaggi (prima la razza, poi la classe) vengono applicati bonus additivi alle caratteristiche, definiti da RaceBonusTable e ClassBonusTable. Con le tabelle di default attive la somma finale è totalPoints + bonusRazza + bonusClasse.

RazzaBonus (default)ClasseBonus (default)
HUMANSTRENGTH +1, AGILITY +1, INTELLIGENCE +1WARRIORSTRENGTH +2, STAMINA +1
ELFAGILITY +2, INTELLIGENCE +1MAGEINTELLIGENCE +2, CHARISMA +1
ORCSTRENGTH +2, RESISTANCE +1THIEFAGILITY +2, LUCK +1
UNDEADRESISTANCE +2, STAMINA +1RANGERAGILITY +2, RESISTANCE +1

Le tabelle si sovrascrivono con raceBonusTable(RaceBonusTable) e classBonusTable(ClassBonusTable); una tabella vuota (builder().build()) è l'opt-out dai relativi bonus.

Un bonus su una caratteristica assente fa fallire la generazione. Se un bonus (di razza o di classe) punta a una caratteristica non presente nel personaggio → IllegalStateException (non viene ignorato né aggiunto). Con le tabelle di default attive non puoi quindi generare un personaggio con un sottoinsieme di caratteristiche che escluda i target dei bonus della razza/classe risolte: usa allCharacteristics() oppure una tabella vuota.

Vedere le fasi: verbose()

verbose() (opzionale) fa loggare a generate() i passaggi della generazione su System.out, con prefisso [CharacterGenerator]: razza e classe risolte, nome, caratteristiche, punti, stato dopo la distribuzione e soprattutto quali e quanto bonus vengono applicati per razza e per classe. È solo per capire cosa succede: senza verbose() non stampa nulla e i valori generati non cambiano.

        .allCharacteristics().totalPoints(20).verbose().generate();

✨ Buff & Debuff

BuffDebuffGeneratorTool genera un set di bonus alle caratteristiche calibrato sulla rarità. L'unica cosa obbligatoria è la rarità.

Uso base
BuffDebuffResult result = BuffDebuffGeneratorTool.building()
        .rarity(Rarity.RARE)
        .generate();

result.buffs();     // es. [BuffElement(STRENGTH, 5), BuffElement(LUCK, 3)]
result.debuffs();   // oggi sempre vuota (vedi nota)

Più alta è la rarità, più bonus (o bonus più forti) possono comparire. Le caratteristiche interessate sono sempre distinte e i valori ricadono nell'intervallo previsto dalla combinazione estratta. Ecco le combinazioni di default per rarità (numero di buff × intervallo valori):

RaritàCombinazioni possibili (count × [min–max])
COMMON1 × [1–2]
UNCOMMON1 × [3–4] · 2 × [1–2]
RARE1 × [5–6] · 2 × [3–4] · 3 × [1–2]
EPIC1 × [7–8] · 2 × [5–6] · 3 × [3–4] · 4 × [1–2]
LEGENDARY1 × [9–10] · 2 × [7–8] · 3 × [5–6] · 4 × [3–4] · 5 × [1–2]

🗺️ Dungeon

DungeonGenerationTool costruisce una mappa di stanze collegate e vi distribuisce eventi principali, nemici, trappole e scrigni. Non usa rarità né buff/debuff, ma segue lo stesso schema building() … generate() e restituisce un DungeonResult. L'unica cosa obbligatoria è il numero di stanze.

Casi tipici
// dungeon minimo: solo il numero di stanze
DungeonGenerationTool.building().numberOfChambers(5).generate();

// con evento finale, evento in posizione casuale, nemici, scrigni e trappole
DungeonResult dungeon = DungeonGenerationTool.building()
        .numberOfChambers(6)
        .mainEvent()                  // codice automatico, nella stanza finale
        .randomPositionMainEvent()    // codice automatico, stanza casuale (non l'ingresso)
        .numberOfEnemy(10)
        .numberOfChests(3)
        .haveTraps()
        .generate();

Le stanze

numberOfChambers(int) è obbligatorio e deve essere almeno 2 (ingresso + finale); mancante o inferiore a 2 → IllegalStateException. La stanza 0 è sempre ENTRY, l'ultima FINAL, le intermedie STANDARD (vedi ChamberType).

Gli eventi principali

Un MainEvent è identificato da un code. Puoi lasciare che il tool generi un codice progressivo (MainEvent_1, MainEvent_2…) o passarne uno esplicito. Ogni metodo è invocabile più volte: ogni chiamata aggiunge un evento.

MetodoDove finisce l'evento
mainEvent() · mainEvent(String code)Sempre nella stanza finale
randomPositionMainEvent() · randomPositionMainEvent(String code)Stanza casuale, diversa dall'ingresso

Nemici, scrigni e trappole

Sono contatori distribuiti uniformemente tra le stanze (base uguale per tutte, il resto a stanze casuali):

Metodo del builderA cosa serveDefault
numberOfEnemy(int)Totale nemici da distribuire (negativo → eccezione)0
numberOfChests(int)Totale scrigni da distribuire (negativo → eccezione)0
haveTraps()Opt-in: genera un numero casuale di trappole in [0, numberOfChambers], poi distribuitonessuna trappola

Le connessioni

Il tool costruisce prima uno spanning tree casuale che garantisce la raggiungibilità di ogni stanza, poi aggiunge fino a numberOfChambers / 2 connessioni extra casuali. Ogni ChamberConnection è normalizzata (from = min, to = max) e deduplicata, quindi A→B e B→A sono la stessa connessione.

Disegnare la mappa: DungeonAsciiRenderer

DungeonAsciiRenderer.render(DungeonResult) è un renderer statico (non un *Tool) che restituisce la mappa come String ASCII: stanze come box con corridoi, glifi </> per ingresso e finale e un'etichetta di stats per stanza, affiancata a una legenda. render(null) → IllegalArgumentException.

SimboloSignificato
< · >Stanza d'ingresso · stanza finale
!NN eventi principali
eNN nemici
^NN trappole
$NN scrigni
#<id>Id della stanza
Render ASCII
String map = DungeonAsciiRenderer.render(dungeon);
System.out.println(map);

🎲 Dadi

DiceLauncherTool tira gruppi di dadi. È l'unico tool che chiude con roll() invece di generate(): non produce un elemento di dominio ma un tiro. Aggiungi uno o più gruppi (es. 2d6 e 1d20), poi roll().

Casi tipici
// un solo gruppo: 3 dadi a 6 facce
DiceLauncherTool.building().dice(3, 6).roll();

// più gruppi, uno con un codice/etichetta
DiceRollResult result = DiceLauncherTool.building()
        .dice(2, 6)
        .dice(1, 20, "attacco")
        .roll();

result.total();   // somma di tutti i dadi di tutti i gruppi

Ogni faccia è pescata in [1, numberOfFaces]. Il subtotal di un gruppo è la somma dei suoi dadi; il total del result è la somma di tutti i subtotali. Il code è opzionale ed è solo un'etichetta per ritrovare il gruppo nel result.

Validazione immediata. All'aggiunta del gruppo, numberOfDice < 1 o numberOfFaces < 2 → IllegalArgumentException. E roll() senza alcun gruppo → IllegalStateException.

Tutte le opzioni del builder

Metodo del builderA cosa serveObbligatorio
dice(int numberOfDice, int numberOfFaces)Aggiunge un gruppo di dadiSì · almeno un gruppo
dice(int numberOfDice, int numberOfFaces, String code)Come sopra, con etichettaSì · almeno un gruppo
roll()Tira i dadi e produce il DiceRollResult—

Risultato: DiceRollResult(List<DiceRoll> rolls, int total), dove DiceRoll(int numberOfDice, int numberOfFaces, String code, List<Integer> results, int subtotal). Solo DiceRollResult implementa GeneratedElementResult; DiceRoll è un elemento interno al result.

Estendere il toolkit

  • Nuovi nomi: aggiungi una riga al file .txt della razza in src/main/resources/namegenerator/. Nessun codice da toccare.
  • Nuova razza: aggiungi una costante a Race con il suo file di nomi e un simbolo.
  • Regole di bonus diverse: implementa BuffDebuffRules e passala con rules(...).
  • Nuovo generatore: prendi JewelGeneratorTool (o WeaponGeneratorTool) come modello — WeaponGeneratorTool e ArmourGeneratorTool ne sono la copia 1:1, cambia solo l'enum del tipo generato.

Cosa NON fa (di proposito)

Per restare semplice e prevedibile, il toolkit non:

  • usa database o configurazione esterna — le liste .txt sono l'unica origine dati;
  • espone un seme casuale: la generazione è volutamente non riproducibile;
  • mantiene stato tra le chiamate: ogni generate() è indipendente.

È tutto qui. building(), qualche opzione, generate(). Buon gioco.
